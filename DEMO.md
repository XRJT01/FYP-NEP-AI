# RenalAI Prototype — Demo Script

A 5-minute walkthrough that exercises every shipped feature. Each step has a "what to point out" note for live demos.

## Prep (30 seconds)

1. From the prototype folder run a static server:
   ```
   python -m http.server 8000
   ```
2. Open `http://localhost:8000` in Chrome or Edge.
3. The first load seeds: 1 patient (Muhammad Sami Afzal, 24M), 7 historical kidney panels (Oct 2025 → Apr 2026), 3 doctors, 2 reviewed shares, and 2 clinical notes.
4. Open `data/sample-reports/lab-sample-chughtai.html` in another tab — Print → "Save as PDF" or screenshot it as PNG. Keep this file ready for the OCR demo step.

## 0. Auth screen — first impression *(20 s)*

> "First open shows a full-screen sign-in / sign-up / demo selector — feels like any modern SaaS."

- Tabs **Sign In · Sign Up · Try Demo**
- For the live demo, click **Try Demo** → instant log-in as Muhammad Sami Afzal with seeded history
- (Alt path) Click **Sign Up** → fill name, email, age, sex, password ≥8 chars → arrives on a clean welcome dashboard with 4 onboarding cards
- Language toggle works on the auth screen too

After log-in: avatar in topbar opens a user-menu with **Settings · Sign out** options.

## 1. Dashboard — first impression *(20 s)*

> "This is what the patient sees on first open. All numbers come from the live risk engine, not hardcoded."

- Risk score **14**, category **NORMAL**, but the trend sparkline rises — point to the colored point at the right edge.
- Four metric tiles: CKD Risk Score, Creatinine 1.42, eGFR 67.8, 7 reports analyzed.
- Right-side "Mild upward drift" alert — generated by the trend analyzer, not handwritten copy.

## 2. Trends — the diagnostic story *(40 s)*

Click **Trends** in the sidebar.

- Three Chart.js line charts: risk score, creatinine (with 1.3 mg/dL upper-limit dashed line), eGFR (with G3 threshold at 60).
- **Trend Summary card on the right** — slope +1.80 pts/mo, R² 0.97, projected G3 in ~3 months at the current eGFR slope.
- Change the date filter to **Last 3 months** — charts and summary recompute live.

## 3. Upload + OCR + Risk Engine — the core flow *(90 s)*

Click **Upload Report**.

### 3a. Drag-and-drop a real lab image
- Drop the PDF/PNG you saved in step 0.
- Watch the four-step pipeline animate: Validate → OCR (live percent from Tesseract.js running in the browser worker) → AI Risk → Generate.
- The right column fills with the **OCR Confirmation form**: each field has a colored confidence badge (green ≥90, amber 80–90, red <80, blue "Edited" if you touch it). Hover the disclosure under any field to see the OCR snippet that matched.
- If a field is "missing" or low-confidence, fix it inline. Validation is live (border turns red for implausible values).
- Click **Confirm & Analyze**. Toast: "Analysis complete — Normal (14)".

### 3b. Or skip OCR — use Manual Entry
- Click reset (×) on the upload card.
- Fill the right-side **Manual Entry** form (e.g. Cr=1.5, eGFR=58, BUN=24).
- Click **Analyze Manually →** — same persistence, same scoring.

### 3c. Camera capture *(optional)*
- Click reset, then **📷 Use Camera** → grant permission → point at a printed lab → **Capture**. Same flow.

## 4. Analysis — the explanation *(45 s)*

You're auto-routed to **Analysis Result** after confirming.

- Risk gauge with KDIGO stage badge ("G2/A2"), model confidence, BUN/Cr ratio.
- **"Why this score" card** — bar chart of per-feature contributions, sorted descending, color-coded.
- **AI Explanation card** — bilingual paragraph with `{cr}`, `{bun}`, `{egfr}`, `{stage}` substitution. Click the **اردو / EN** toggle: in-place flip, no full page re-render, Nastaliq font + RTL applied automatically.
- **Personalized Recommendations** — numbered list. The number turns red for the `urgent` action when the category is CRITICAL.

## 5. PDF Export *(20 s)*

Still on Analysis Result, click **⇩ Export PDF**.

- A4 PDF lands in Downloads named `RenalAI-2026-MM-DD.pdf`.
- Open it: header → risk assessment → biomarker table → contributions → English explanation → Urdu explanation (Nastaliq renders correctly) → recommendations → disclaimer.

## 6. History — drill-in + bulk export *(40 s)*

Click **Report History** in the sidebar.

- 8 rows now (7 seeded + the one you just analyzed).
- Click any column header to sort (e.g. **Score ↓** to sort high to low).
- Toggle a category chip (e.g. turn off **Normal**) to filter.
- Click any row → drilled into that historical analysis. Note the "Historical view" badge in the title and the **↻ Show latest** button.
- Back to History → click **⇩ Export All as PDF** → multi-page document with cover page + per-report sections, all separated by page breaks.

## 7. Doctor Portal *(60 s)*

Click **Doctor Portal**.

### 7a. Patient view
- **My Sharing** tab is default. Existing share entries: Dr. Tayyab (Reviewed, with urgent note), Dr. Sara (Reviewed).
- **Right column** shows incoming clinical notes — Dr. Tayyab's urgent note has a red border.
- Try to share again with the same doctor → validation error "already shared".
- Try a fake doctor ID `DOC-XXXX-XXXX` → "Doctor not found".
- Use a valid ID from the autocomplete (`DOC-7E2D-AR18` for Dr. Ahmed Raza), pick a report, hit **Share Report ✦** → toast confirms, new entry appears in My Doctors.

### 7b. Doctor view
- Click the **Inbox (Doctor view)** tab.
- "Acting as" picker — switch between the 3 seeded doctors. Each has different visible inbox content based on what was shared with them.
- Pick **Dr. Tayyab Khushnood** → his inbox shows the patient's Apr 27 share with the urgent note already attached.
- Type a follow-up note in the composer, tick **Mark as urgent**, click **Add Clinical Note** → toast "Clinical note added", note appears in red-bordered card.
- Switch back to the patient tab — the new note is visible immediately.

## 8. Settings + Privacy *(45 s)*

Click **Settings**.

### 8a. Language
- Open the Language dropdown → pick **اردو (Urdu)** → entire UI flips to RTL Nastaliq.
- Switch back to English.

### 8b. Notifications
- Toggle the three switches. Each emits a toast. State persists across reloads.

### 8c. Export My Data
- Click **⇩ Export My Data** → JSON file lands in Downloads with the complete bundle: user, reports, biomarkers, analyses, shares, notes, doctors, audit. This is the "data portability" GDPR requirement satisfied.

### 8d. Audit log
- Scroll to the bottom **Recent Activity** card.
- Filter dropdown — pick `Exported PDF` to see only the PDFs generated this session, with timestamps.
- Switch back to **All actions**. Every share / analysis / note / export this session is logged.

### 8e. Delete All Data *(skip in real demo unless re-seeding)*
- Click **✕ Delete All Data** → modal confirmation → wipes IndexedDB → page reloads → you start fresh with the original seed.

## What to highlight in a Q&A

| Question | Answer |
|---|---|
| Is OCR really running in the browser? | Yes — Tesseract.js v5 in a Web Worker. Open DevTools console, you'll see `recognizing text 47%` log lines. |
| Where does the risk score come from? | A KDIGO-grounded weighted ensemble in `js/risk-engine.js`. CKD-EPI 2021 race-free formula auto-fills eGFR when only creatinine is present. The contributions card on the analysis page exposes per-feature contribution for explainability. |
| What about privacy? | Zero network requests for any patient data. Everything lives in IndexedDB on this device. Image previews use object URLs that are revoked immediately after analysis. The seed marks all reports with a `deletedAt` timestamp to mirror the FYP report's "image auto-deleted within 60 minutes" policy. |
| Can it work offline? | The PWA manifest is wired so the app is installable. A service worker + offline upload queue is on the roadmap (out of v1 scope). |
| How would this become the FastAPI/PostgreSQL stack from the report? | All persistence goes through `RAStorage` — swap that single module to call FastAPI endpoints and the rest of the codebase needs no changes. The risk engine, OCR pipeline, and recommendations would either move server-side (production) or stay client-side (offline-first variant). |
